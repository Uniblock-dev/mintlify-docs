# What is Unified API?

Unified API provides a single, consistent interface to query blockchain data across 100+ networks. Instead of learning different API patterns for each chain, you use the same endpoints for tokens, NFTs, transactions, and market data—regardless of the underlying blockchain.

**Use Unified API when you need to:**
- Query token balances and prices across multiple chains
- Track NFT collections, metadata, and floor prices
- Monitor transaction history and wallet activity
- Access real-time market data and trading volumes
- Scan blockchain networks for contracts and events

<Info>
  Get started in minutes. Create your project and API key at [dashboard.uniblock.dev](https://dashboard.uniblock.dev).
</Info>

## Why Unified API?

### One API, all chains

Write your integration once and it works across Ethereum, Polygon, Solana, BSC, Arbitrum, Base, and 100+ other chains. No need to manage multiple provider SDKs or learn chain-specific quirks.

```bash Example: Same endpoint works on any chain
# Ethereum (chainId=1)
curl 'https://api.uniblock.dev/uni/v1/token/balance?chainId=1&address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb' \
--header 'x-api-key: YOUR_API_KEY'

# Polygon (chainId=137)
curl 'https://api.uniblock.dev/uni/v1/token/balance?chainId=137&address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb' \
--header 'x-api-key: YOUR_API_KEY'

# Solana (chainId=solana)
curl 'https://api.uniblock.dev/uni/v1/token/balance?chainId=solana&address=7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU' \
--header 'x-api-key: YOUR_API_KEY'
```

### Built-in reliability

Uniblock automatically routes requests to the best available provider and retries failed requests with backup providers. You get enterprise-grade reliability without managing infrastructure.

### Optimized for performance

Auto-routing selects providers based on latency, cost, and reliability. Requests are optimized for speed and cost-efficiency automatically.

## Available APIs

### Token API

Query fungible token data including balances, prices, transfers, and allowances across all supported chains.

**Common use cases:**
- Display wallet token balances
- Track token price changes
- Monitor token transfers
- Check spending allowances

**Key endpoints:**

| Endpoint | Description |
|----------|-------------|
| `token/balance` | Get current token balances for an address |
| `token/price` | Get real-time USD price of a token |
| `token/price/historical` | Get historical price data |
| `token/transfers` | Get token transfer history |
| `token/allowances` | Get approved spending allowances |
| `token/metadata` | Get token name, symbol, decimals |

[Explore Token API →](/reference/unified-api-reference-overview#token-api)

### NFT API

Access comprehensive NFT data including collections, metadata, floor prices, sales, and ownership across all chains.

**Common use cases:**
- Build NFT marketplaces
- Display user NFT portfolios
- Track collection floor prices
- Monitor NFT sales and transfers

**Key endpoints:**

| Endpoint | Description |
|----------|-------------|
| `nft/balance` | Get NFTs owned by an address |
| `nft/metadata` | Get NFT metadata and attributes |
| `nft/floor-price` | Get collection floor prices by marketplace |
| `nft/transfers` | Get NFT transfer history |
| `nft/sales` | Get NFT sales from on-chain marketplaces |
| `nft/collection-metadata` | Get collection info and stats |

[Explore NFT API →](/reference/unified-api-reference-overview#nft-api)

### Transaction API

Query transaction history, receipts, and detailed transaction data for any address or block.

**Common use cases:**
- Display transaction history
- Track wallet activity
- Analyze transaction patterns
- Build block explorers

**Key endpoints:**

| Endpoint | Description |
|----------|-------------|
| `transactions` | Get transactions for an address |
| `transaction` | Get specific transaction details |
| `transactions/block` | Get all transactions in a block |
| `transactions/advanced` | Get detailed transaction data with token/NFT transfers |

[Explore Transaction API →](/reference/unified-api-reference-overview#transaction-api)

### Market Data API

Access real-time and historical market data including prices, market caps, volumes, and trending tokens.

**Common use cases:**
- Display token prices in your app
- Build price charts and analytics
- Track trending tokens
- Monitor market movements

**Key endpoints:**

| Endpoint | Description |
|----------|-------------|
| `market-data/price` | Get real-time token prices |
| `market-data/market-cap` | Get current market capitalization |
| `market-data/24-hour-volume` | Get 24-hour trading volume |
| `market-data/chart-range` | Get price/volume data for charts |
| `market-data/trending` | Get trending tokens (gainers, losers, most searched) |

[Explore Market Data API →](/reference/unified-api-reference-overview#market-data-api)

### Scans API

Access blockchain explorer functionality including contract verification, gas estimation, and event logs.

**Common use cases:**
- Verify smart contracts
- Estimate gas prices
- Query event logs
- Get contract ABIs

**Supported networks:**
- Etherscan (Ethereum)
- Polygonscan (Polygon)
- BscScan (Binance Smart Chain)

**Key endpoints:**

| Endpoint | Description |
|----------|-------------|
| `scan/current-gas-estimation` | Get current gas prices (safe, proposed, fast) |
| `scan/contract-abi` | Get verified contract ABI |
| `scan/contract-source-code` | Get verified contract source code |
| `scan/account-normal-transactions` | Get transaction list for an address |
| `scan/log-address` | Get event logs from a contract |

[Explore Scans API →](/reference/unified-api-reference-overview#scans-api)

### Stellar API

Access Stellar Horizon REST API for both mainnet and testnet networks.

**Use cases:**
- Query Stellar accounts and balances
- Track Stellar transactions and operations
- Access ledger and payment data

[View Stellar Overview →](/reference/unified-api/overview-stellar)

## Common parameters

All Unified API endpoints share consistent parameter patterns:

<ParamField query="chainId" type="string" required>
  Network identifier. Use numeric IDs for EVM chains (e.g., `1` for Ethereum, `137` for Polygon) or string identifiers for non-EVM chains (e.g., `solana`, `stellar`).
  
  [View all supported chains →](/reference/resources/chains/supported-chains)
</ParamField>

<ParamField query="address" type="string" required>
  Wallet or contract address. Format varies by chain:
  - **EVM chains**: 0x-prefixed hex (e.g., `0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb`)
  - **Solana**: Base58 encoded (e.g., `7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU`)
  - **Stellar**: G-prefixed (e.g., `GDAYVCINVNUZ57EOCN4FK2VVWGQ3L3NW37L6UJLZCK3C7S7CNSS5EHHG`)
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of results to return. Most endpoints support pagination.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip for pagination.
</ParamField>

## Response format

All Unified API responses follow a consistent structure:

```json Success response
{
  "data": [...],  // Array of results or single object
  "meta": {       // Optional metadata
    "total": 150,
    "limit": 100,
    "offset": 0
  }
}
```

```json Error response
{
  "error": {
    "code": "INVALID_CHAIN_ID",
    "message": "Chain ID '999' is not supported",
    "details": {
      "chainId": "999"
    }
  }
}
```

## Common pitfalls

<Warning>
  **Invalid chain IDs** - Always verify the chain ID is correct. Use numeric IDs for EVM chains and string identifiers for non-EVM chains. See [supported chains](/reference/resources/chains/supported-chains).
</Warning>

<Warning>
  **Address format mismatches** - Ensure addresses match the chain's format. EVM addresses must be 0x-prefixed, Solana addresses are base58, etc. Invalid formats return validation errors.
</Warning>

<Warning>
  **Rate limits** - Free tier has rate limits. Monitor usage in your dashboard and upgrade if needed. See [pricing plans](/reference/resources/pricing-plans).
</Warning>

<Warning>
  **Pagination** - Large result sets require pagination. Use `limit` and `offset` parameters to fetch all results. Default limits vary by endpoint.
</Warning>
